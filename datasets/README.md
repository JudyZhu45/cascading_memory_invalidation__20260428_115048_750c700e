# Datasets

This directory contains datasets for the research project on Cascading Memory Invalidation in User Memory Graphs. Data files are NOT committed to git due to size. Follow the download instructions below.

---

## Dataset 1: LoCoMo (Long-term Conversational Memory)

### Overview
- **Source**: `Aman279/Locomo` on HuggingFace (mirrors original `snap-research/locomo`)
- **Size**: 35 dialogue samples, avg 300+ turns per conversation (9K tokens avg)
- **Format**: HuggingFace Dataset (parquet), features: `dialogue_id`, `turns` (JSON string with `speaker_role`, `utterance`)
- **Task**: Long-term conversational memory evaluation (QA, event summarization, dialog generation)
- **Splits**: train (35 samples)
- **License**: See original dataset

### Download Instructions

**Using HuggingFace (recommended):**
```python
from datasets import load_dataset
import os
ds = load_dataset('Aman279/Locomo')
os.makedirs('datasets/locomo', exist_ok=True)
ds.save_to_disk('datasets/locomo')
```

**Loading once downloaded:**
```python
from datasets import load_from_disk
ds = load_from_disk('datasets/locomo')
train = ds['train']
import json
turns = json.loads(train[0]['turns'])
print(turns['speaker_role'][:5])
print(turns['utterance'][:3])
```

### Notes
- Original paper: Maharana et al. (2024), arXiv:2402.17753
- Contains multi-session conversations spanning months with persona and temporal event graph grounding
- Human annotators verified long-range consistency
- Key for evaluating memory retrieval methods (QA tasks with single-hop, multi-hop, temporal reasoning)

## Dataset 2: HorizonBench (Long-Horizon Preference Tracking)

### Overview
- **Source**: `stellalisy/HorizonBench` on HuggingFace
- **Size**: 4,245 benchmark items from 360 simulated users
- **Format**: HuggingFace Dataset, features: `id`, `generator`, `user_id`, `conversation`, `correct_letter`, `options`, `has_evolved`, `preference_domain`, `distractor_letter`, `preference_evolution`
- **Task**: Long-horizon personalization — identifying current user preference from 6-month conversation history (5-option multiple choice)
- **Splits**: test (4,245 items)
- **License**: See stellalisy/HorizonBench

### Download Instructions

**Using HuggingFace (recommended):**
```python
from datasets import load_dataset
import os
ds = load_dataset('stellalisy/HorizonBench')
os.makedirs('datasets/horizonbench', exist_ok=True)
ds.save_to_disk('datasets/horizonbench')
```

**Loading once downloaded:**
```python
from datasets import load_from_disk
ds = load_from_disk('datasets/horizonbench')
test = ds['test']
print(test[0])
print('has_evolved:', test[0]['has_evolved'])
print('preference_domain:', test[0]['preference_domain'])
```

### Key Features
- `has_evolved`: Boolean — whether the target preference changed via a life event
- `distractor_letter`: The option corresponding to the pre-evolution value (for analyzing belief-update failure)
- `preference_evolution`: Provenance of the preference change (which life event caused it)
- 59% evolved preferences, 41% static
- Generated by three frontier generators: Claude-sonnet-4.5 (1,052), o3 (981), Gemini-3-flash (2,212)
- Context window: ~163K tokens avg (up to 250K+)

### Notes
- Key paper: HorizonBench (arXiv:2604.17283), Li et al. 2026
- Uses structured mental state graph with typed dependency edges — directly relevant to this research
- Best frontier model (Claude-opus-4.5) achieves only 52.8%; most score at or below 20% chance baseline
- Pre-evolution distractor selected >35% of wrong answers (1.5× uniform baseline)
- **Primary evaluation dataset for this research**

## RippleEdits Benchmark (included in code/)

The RippleEdits benchmark data is available in `code/RippleEdits/data/benchmark/`:
- `recent.json`: Recent entity edits
- `random.json`: Random entity edits  
- `popular.json`: Popular entity edits

Each entry contains a factual edit and 6 evaluation criteria (Logical Generalization, Compositionality I & II, Subject Aliasing, Preservation, Relation Specificity) for evaluating ripple effects.

**Loading:**
```python
import json
with open('code/RippleEdits/data/benchmark/recent.json') as f:
    recent_edits = json.load(f)
print(f'Number of edits: {len(recent_edits)}')
print('First edit:', recent_edits[0]['edit_id'])
```
